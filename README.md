# Power Risk Lab

Power Risk Lab is a research system for weather-driven electricity demand and power-market risk modeling under nonstationary conditions.

The current scaffold focuses on a first reproducible data path:

```text
EIA-930 PJM hourly load
ERA5 hourly weather over a broad PJM box
joined exploratory dataset
minimal modeling feature table
```

The project prioritizes evaluation integrity, temporal correctness, uncertainty, regime-aware diagnostics, and reproducible research workflows.

## Current Status

Implemented:

- config-driven source pipelines for EIA-930 and ERA5
- config-driven joined dataset construction
- config-driven minimal feature generation
- exploratory notebooks for load, weather, joined data, and feature/load diagnostics
- pre-commit hygiene configuration

Current sample limitation:

- ERA5 weather currently covers only `2023-01-01` through `2023-01-07`.
- The minimal feature config includes a `168h` load lag plus a `1h` target horizon, so the strict modeling output may contain zero rows until a longer weather window is built.

## Setup

Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Install pre-commit hooks:

```bash
pre-commit install
pre-commit run --all-files
```

Credentials:

- EIA API key: keep in `.env` or shell environment as `EIA_API_KEY`.
- Copernicus CDS credentials: keep in `~/.cdsapirc`.

Do not commit credentials or data artifacts.

## Data Pipelines

Build PJM EIA-930 load:

```bash
python -m src.data.eia930 --config configs/data/sources/eia930_pjm_2023.yaml
```

Download EIA-930 raw data before cleaning:

```bash
python -m src.data.eia930 --config configs/data/sources/eia930_pjm_2023.yaml --download
```

Build ERA5 PJM-box weather features:

```bash
python -m src.data.era5 --config configs/data/sources/era5_pjm_box_2023_0101_0107.yaml
```

Build the joined exploratory dataset:

```bash
python -m src.data.datasets --config configs/data/datasets/pjm_load_era5_sample.yaml
```

Build the minimal modeling feature table:

```bash
python -m src.features.build_features --config configs/features/pjm_load_weather_minimal.yaml
```

## Research Guardrails

- Use walk-forward validation for time series.
- Avoid random train/test splits.
- Preserve missingness and data lineage.
- Treat same-hour realized weather as descriptive unless the forecast design permits it.
- Compare against persistence and seasonal naive baselines.
- Evaluate by regime, not only average error.
